# Polymarket's response to macro shocks: the SVB collapse case study

NeurIPS-formatted workshop-style case study of how Polymarket repriced 20
markets (SVB-direct, contagion, FOMC outcome grid) around the March 2023
banking shock. Every number in `main.tex` is produced by the scripts in
`scripts/`; if a number is not in `data/numbers.txt` after a clean run, it
is not in the paper.

## How to reproduce the numbers in this paper

```bash
cd papers/polymarket-svb-collapse-macro-shock-response
python3 -m venv .venv && source .venv/bin/activate  # optional
pip install -r scripts/requirements.txt
python3 scripts/fetch_data.py   # pulls Polymarket Gamma + CLOB and Yahoo Finance
python3 scripts/analysis.py     # produces data/numbers.json + data/numbers.txt
python3 scripts/figures.py      # renders figures/*.pdf
```

`fetch_data.py` is the only script that hits the network. It writes ~2 MB
into `data/` (Polymarket CSVs and Yahoo CSVs). `analysis.py` and
`figures.py` read those CSVs only, so once you have the cached data the
rest is fully offline.

To compile the paper you also need a working LaTeX install (the task
container that produced this paper did not have one; the source compiles
under TeX Live 2024 with the bundled `neurips_2024.sty`).

## What's checked in

- `main.tex`, `references.bib`, `neurips_2024.sty` — the paper source
- `scripts/fetch_data.py`, `analysis.py`, `figures.py`, `requirements.txt`
- `figures/*.pdf` + `*.png` — committed so the paper builds without
  re-running the pipeline
- `data/numbers.txt`, `data/numbers.json` — committed so a reviewer can
  spot-check the paper's numbers without running anything

## What's not checked in

- `data/prices/`, `data/yahoo/`, `data/candidates.json` — regenerated by
  `fetch_data.py`. About 2 MB.
- `.venv/`, `__pycache__/`, OS junk — see `.gitignore`.
